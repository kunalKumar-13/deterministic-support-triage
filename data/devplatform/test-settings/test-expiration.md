# Test expiration and active period

Tests in DevPlatform remain active indefinitely unless a start and end
date/time are configured. Without these fields populated, tests do not
expire automatically and continue to accept new invitations.

## Setting an expiration window

Go to the test's **Settings > General** section and set the **Start date &
time** and **End date & time** fields.

After the configured end time:

- Invited candidates can no longer access the test.
- The **Invite** button is disabled; no new invitations can be sent.
- Existing reports and submitted attempts remain available for review.

## Keeping a test active indefinitely

To remove an expiration that was previously configured, open
**Settings > General** and clear the Start/End fields by clicking the
clear icon (X). Tests with no start/end set never expire.

## Common causes of "candidates not receiving new tests"

- A previously-configured expiration has already passed.
- The candidate is on a deactivated workspace seat.
- The test was archived (separate from expiration).
